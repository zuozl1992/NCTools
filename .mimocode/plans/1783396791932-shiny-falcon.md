# MySSCOM 重构计划

## 目标
将 MySSCOM 串口网络调试工具从单体 Qt Widgets 应用重构为模块化、界面与逻辑分离的 QML 应用，基于 Qt 6.8.3。

## 3. 核心层设计

### 3.1 AbstractTransport 抽象接口

```cpp
// transporttype.h
enum class TransportType { Serial, TcpServer, TcpClient, Udp };

// transportconfig.h
struct SerialConfig {
    QString portName;
    qint32 baudRate;
    QSerialPort::DataBits dataBits;
    QSerialPort::StopBits stopBits;
    QSerialPort::Parity parity;
    QSerialPort::FlowControl flowControl;
};

struct TcpServerConfig { quint16 listenPort; };
struct TcpClientConfig { QString host; quint16 port; };
struct UdpConfig { quint16 localPort; QString peerHost; quint16 peerPort; };

using TransportConfig = std::variant<SerialConfig, TcpServerConfig, TcpClientConfig, UdpConfig>;

// abstracttransport.h
class AbstractTransport : public QObject {
    Q_OBJECT
    Q_PROPERTY(bool connected READ isConnected NOTIFY connectedChanged)
    Q_PROPERTY(QString errorString READ errorString NOTIFY errorOccurred)
public:
    explicit AbstractTransport(QObject *parent = nullptr);
    ~AbstractTransport() override;

    virtual bool open() = 0;
    virtual void close() = 0;
    virtual bool isConnected() const = 0;
    virtual bool sendData(const QByteArray &data) = 0;
    virtual QString errorString() const = 0;

signals:
    void dataReceived(const QByteArray &data);
    void connectedChanged();
    void errorOccurred(const QString &message);
};
```

### 3.2 具体传输实现

| 类 | 职责 | 从原代码提取 |
|---|---|---|
| `SerialTransport` | 串口通信封装 | `initPort()`, `sendUseSerial()`, `closeSerialport()`, `serialReadyReadSlot()` |
| `TcpServerTransport` | TCP 服务器封装 | `sendUseTcpServer()`, `closeTcpServer()`, `tcpServerNewConnectionSlot()`, `tcpServerSocketReadyReadSlot()` |
| `TcpClientTransport` | TCP 客户端封装 | `sendUseTcpClient()`, `closeTcpClient()`, `tcpClientReadyReadSlot()`, `tcpClientDisconnectedSlot()` |
| `UdpTransport` | UDP 通信封装 | `sendUseUdp()`, `closeUdp()`, `udpReadyReadSlot()` |

## 11. 风险与缓解

| 风险 | 缓解措施 |
|------|---------|
| QML 与 Widgets 行为差异 | 逐个功能对比测试，保持交互逻辑一致 |
| 信号槽连接遗漏 | 使用测试覆盖所有数据流路径 |
| 设置迁移兼容 | 保持 INI 格式不变，直接复用 QSettings |
| 串口虚拟对测试困难 | macOS/Linux 使用 pty，Windows 使用 com0com |
| 文件发送性能 | 保持 1024 字节分块，添加进度回调 |
